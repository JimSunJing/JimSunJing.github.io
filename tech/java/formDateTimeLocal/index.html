前端表单传入时间(datetime-local)格式无法被 Spring MVC 处理的解决方案

情况

在做一个添加条目的页面时，使用到了一个 type="datetime-local" 的时间input，结果导致提交表单后服务器端报错 400: The request sent by the client was syntactically incorrect.

原因

通过chrome抓包，得知传入的 time 字符串的格式为：“2020-02-20T13:13”，也就是：“yyyy-MM-ddTHH:mm”

我认为这是因为MVC自带的格式转换器没有该字符串转时间类型的转换器。

由于不需要自建和配置，对于个别出现的成员比较方便，但对于需要经常转换的格式还是用converter更加合适。
